# Plans

This folder holds implementation plans produced by the **product-owner** agent (Phase 1). Plans are saved here as `YYYY-MM-DD-short-feature-name.md` so Phase 2 can be invoked with a path, e.g.:

- Re-invoke the product-owner agent with: **Approved**, plan: `docs/plans/2025-03-08-save-games.md`.

Plans in the parent `docs/` folder (e.g. `WEBRTC_IMPLEMENTATION_PLAN.md`) are project docs; plans here are approval-gated outputs from the PO workflow.
